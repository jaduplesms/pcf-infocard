Key tips:
• Use address1_composite (the composite address field) for detail slots — NOT individual address fields (city, street, zip). The composite field gives you "191 Common Lane, Pyrford KT14 7HT" as a single value.
• The same field CAN be bound to multiple slots — e.g. telephone1 in both phoneField1 (action icon) and detailField2 (visible number).
• Empty slots are simply ignored — no need to bind all 28.
• Action slots auto-hide when the bound field is empty (e.g. no email → no email icon).
