Option A: Contact Bar Card

Account name as header (no label). Phone/website/email as a tappable contact bar with labels. Remaining fields below in compact rows. Empty fields hidden entirely.

~160px — 7 fields shown, 3 empty hidden, contact info actionable
Design rationale:
  • Account name is the card header — no label needed
  • Account number as subtitle — always visible, low prominence
  • Phone shows number (tappable to call), website is icon-only (tappable to open)
  • Email/Fax/Other Phone hidden because they're empty
  • Remaining metadata fields in compact label:value rows
  • If email had a value, it would appear as a mail icon in the contact bar

Option B: Icon Row + Tags

Action icons top-right. Key details as icon+text rows. Metadata as small tags at the bottom. Maximum density.

~130px — contact-card feel, tags for metadata
Design rationale:
  • Feels like a native contact card — familiar UX for mobile users
  • Phone + website as icon buttons top-right (instant tap to call/open)
  • Address and phone as icon+text rows (Google Maps style)
  • Low-priority metadata (industry, SIC, status) as compact tags
  • Most compact option — everything visible at a glance
  • Phone appears in both the icon button AND the detail row (debatable — could remove one)

Option C: Two-Column Grid

Dense data grid with paired fields side-by-side. Action icons in the header. Good for accounts with more data fields.

  • Pairs shorter fields (account # + industry, phone + SIC) side by side
  • Address spans full width since it's longer
  • Tiny uppercase labels above values — minimal space overhead
  • Scales well when there are 8-12+ fields (just add more grid cells)
  • Action icons in header, same as Option B
