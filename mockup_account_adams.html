What's shown: Account name, primary contact + territory as subtitle, phone (tappable), website (icon), address with pin icon, currency + territory in grid.

What's hidden (empty): Email, Fax, Parent Account, Billing Account, Product Price List, Work Order Instructions.

Key choices:
  • Primary Contact promoted to subtitle — most useful "at a glance" info for a tech
  • Territory appears twice (subtitle + grid) — could remove from grid
  • Phone shown with number visible (techs need to see it, not just tap)
  • Address collapsed to one line (from 3 lines + section header)

Trade-offs:
  • Phone number visible in body AND tappable via icon — tech can see it or call it
  • Currency just shows "GBP" tag instead of "British Pound" (is that enough?)
  • No explicit "Currency" or "Territory" labels — relies on context (tag color, icon)
  • Drops Work Order Instructions entirely when empty — if populated, could expand card
  • Least formal — feels like a contact card, not a form

Option C: Sectioned Compact

Preserves the section structure (contact / details / address) but collapses each dramatically. Good middle ground if stakeholders want to keep the "form feel."
