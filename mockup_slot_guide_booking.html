Future: Related Entity Fields

Fields from the Work Order's Service Account (customer phone, address) require a webAPI.retrieveRecord() call. Confirmed working online and offline.

Planned: relatedFields config property
A text config property where the developer specifies which lookup to follow and which fields to fetch. Example format:

msdyn_workorder.msdyn_serviceaccount → subtitle3
msdyn_workorder.msdyn_primaryincidenttype → tag2
msdyn_workorder.address → detail1
The control reads the lookup ID from the bound field, calls webAPI.retrieveRecord() to fetch the specified columns, and injects the results into the target slots. Works offline when the related entity is in the Mobile Offline Profile.
