Decision Guide

Choose the right layout for your scenario.

Smart

The default choice. Balances information density with a clean, progressive-disclosure pattern.

Best for
  • Mobile forms where screen space is limited
  • Field Service technician booking views
  • Forms with many other controls where the card should not dominate
  • Quick-view forms embedded in other records
  • Scenarios where users need the summary at a glance but occasionally need details
Tip: Use startExpanded=false on busy forms to save initial space.

Contact

Shows everything at once. No user interaction needed to see all data.

Best for
  • Desktop forms with ample horizontal space
  • Contact-centric views (Account, Contact entity)
  • Forms where all fields are always relevant
  • Print/export scenarios where collapsed content would be lost
  • Users who prefer no interaction required to see data
Tip: Pair with hideEmptyFields=true to keep the card from looking sparse when optional fields are blank.

Compact

Maximum information density. Every field rendered as a labelled key-value row.

Best for
  • Data-heavy forms like Customer Asset or equipment records
  • Dashboard-style views with multiple cards
  • Forms where labels matter more than visual hierarchy
  • Accessibility scenarios where icons may not convey enough meaning
  • Users accustomed to traditional form layouts
Tip: The compact layout uses section headers (Contact, Details) for visual grouping rather than borders and icons.

Quick Reference: Setting the Layout

The layout is configured via the layout property in the form editor:

Smart
layout = "smart"
Contact
layout = "contact"
Compact
layout = "compact"
